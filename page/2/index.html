多态（Polymorphism）

本文主要探讨：

  • 静态多态（编译时多态）

    方法重载(method overloading)

  • 动态多态(运行时多态)

    方法重写(method overriding)

  • ‘virtual’关键词和虚方法

  • 方法重写和方法隐藏的差异
  • 密封方法（Sealed method）

什么是多态？

多态意味着一个名字多个形式；一个对象多种操作形式；一个函数多种不同形式的使用，就是说一个带有多种形式的对象叫做多态。

Example 3

你的手机，有一个名字，但是有多种形式存在：

  • 作为手机
  • 作为照相机
  • 作为音乐播放器
  • 作为游戏机

通过多态， 根据运行时触发机制，相同方法或者属性会有不同的执行操作。

静态多态

在静态多态中，执行的决定是在编译的时候定下的。
哪个方法将会被将会被执行仅仅是通过编译时决定。
方法的重载就是这样的。

编译时多态就是方法的重载，需要执行的时候，编译器知道哪个重载方法可以执行。

方法重载是：它的的一个类有多个方法有相同的名字和不同的参数

编译器检查传给方法的参数类型以及参数的个数并且决定哪个方法在编译的时候运行。并且在编译时候没有方法满足，将会报错。

动态/运行时多态

运行时多态是通过方法重写实现的。
方法重写允许我们在基类中定义多个方法，并在在其子类中定义相同的方法名次以及相同的参数。

通过运行时多态，我们可以在运行的时候指向任何一个来之基类的对象的子类，并且体现运行时绑定的功能。
（By runtime polymorphism, we can point to any derived class from the object of the base class at runtime that shows the ability of runtime binding.）

通过基类对象的引用，哪个方法被调用是取决于对象是哪个变量正在引用的。

编译器是不知道方法是否可以重写的，所以编译器在编译的时候是不会报任何错误的。在运行的时候，它会决定让哪个方法去调用，如果没有方法，则会报错

编译器请求虚方法 show() 并且成功编译。正确的show()方法的执行一直到到运行的时候决定，并且那个时候基类的objBase是作为子类的初始化的。

Virtual Keyword

根据MSDN,”Virtual” 关键词是用来修改一个方法，属性，indexer甚至一个时间的申明的。它是可以在子类中重写（overriding）的。

虚方法

虚方法是一个它的行为是可以被继承类重写的方法。虚方法容许在基类中申明一个可以被继承类重新定义的方法。

默认的，方法不是虚拟的，你不能重写一个非虚拟的方法。
virtual不能和 static, abstract,private ,override等修饰一起使用

虚属性的行为就像抽象方法，除了在定义和出发的语法上有区别。

在静态的属性上使用virtual关键词是错误的。

通过使用override修饰符，可以重写父类中virtual修饰的属性。

虚方法可以解决如下的问题：
在OOP中，当一个继承类继承自一个基类，一个子类的对象可能被引用作基类类型或者子类类型。如果基类的方法被继承类重写，那么这个方法的执行行为是不确定，模棱两可的。

C#中多态是需要显示的设定的。必须要通过virtual或者abstract关键词来修饰。

如果不放置一个修饰词在基类中，多态是不能产生的。如果添加一个不可修改的方法到继承类中，并且在基类中存在相同的方法，但是该方法是不可以被修改，被重写的，编译器会警告。

它意味着你在隐藏（或者说重新定义）基类的方法。通过以上案例，要特别注意这点。

在其他的语音中，比如Java，Java有个隐形的多态，上面的形式可以在在Java中达到预期的执行。

Java中，一个类的所有方法默认都是虚方法，除非开发者决定使用final关键词来阻止继承来重写那些方法。
相反的，C#采用的是被C++使用的策略：开发者必须主动的使用virtual关键词来给子类重写那些方法。而且C#中所有的方法都是非虚的。

方法重写容许一个子类提供一个特定的实现，来实现那些在已经在基类中定义的方法。子类的实现替换了基类中的实现。

当一个虚方法被引用调用的时候，类中的哪个方法的实现会被调用，这是由那些引用引用的对象的实际类型来决定的。

C#学习笔记(5)

Public vs Private vs protected vs internal

  1. public: 对任何类和成员都公开, 无限制访问;
  2. protected: 仅仅对该类以及该类的派生类公开;
  3. private: 仅仅对该类公开;
  4. internal: 只能值包含该类的程序集中访问该类(只是单独的项目,而不是整个解决方案);
  5. protected internal: 只能在本类,派生类或者包含该类的程序集中访问.

微软官方文档解释：

public

The type or member can be accessed by any other code in the same assembly or another assembly that references it.

private

The type or member can only be accessed by code in the same class or struct.

protected

The type or member can only be accessed by code in the same class or struct, or in a derived class.

internal

The type or member can be accessed by any code in the same assembly, but not from another assembly.

protected internal

The type or member can be accessed by any code in the same assembly, or by any derived class in another assembly.


Only one access modifier is allowed for a member or type, except when you use the protected internal combination.
Access modifiers are not allowed on namespaces. Namespaces have no access restrictions.
Depending on the context in which a member declaration occurs, only certain declared 
accessibilities are permitted. If no access modifier is specified in a member declaration, a default accessibility is used.
Top-level types, which are not nested in other types, can only have internal or public accessibility. The default accessibility for these types is internal.

C#学习笔记(3)

  1. 判断是不是某种类型 x is SomeClass
  2. ClassDog,ClassAnimal,ClassTree. ClassDog extends ClassAnimal. 如果做强制类型转化 dog = (ClassDog)ani, 这样多半没有问题，如果是tree = (ClassTree)dog 则报异常。CSharp中提供了比Java等更加安全的方式 ClassTree tree2 = tree as dog 这样在Runtime的时候会检查类型，类型不对,则tree2 == null.而不会出现异常。
  3. as 必须用作引用类型
  4. 值类型转string。比如：int x = 10; string xstr = x.ToString();即可。
  5. string转成值类型，比如int. int x = Convert.ToInt32(xStr); 比必须使用Convert的方法才行
  6. 如果xStr = “123”,则没有问题，如果是”abc”,则报异常。
  7. 还可以Int32.Parse(xStr)也是可以的
  8. 还有一种安全的string to int 转换： int x; bool suc = Int32.TryParse("abc", out x); 如果成功，则x有正确的值， suc也是true，否则x == 0,suc == false.但是不会报异常，这是很安全的处理方法
  9. 双??的用法,比如 int x = xx ?? 500; 表示如果 xx 是空值，则 x == 500 ,否则 x == xx

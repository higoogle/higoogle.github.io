List vs ArrayList

IList and List can only store same type values, but ArrayList can store diff types, For example:

解释一下：

  1. List<T>由于是强类型的，所以没有拆箱，装箱的转换
  2. ArrayList它是可以添加任何的类型的，它做安全检查，所以在插入的时候，性能是最好的，但是取值的时候就会比较的麻烦
  3. ArrayList是C#在没有泛型的是存在的，有泛型后，就不推荐使用ArrayList的了。

所以呢，是否一定选择使用List而不使用ArrayList，也不是一概而论的,因为大多数情况下，可能集合里面存放的值的个数是很少的，那么ArrayList性能就比较的好了，只要编程的时候注意，即使使用ArrayList也要保证存放的值都是同一种类型的，还是可以大大发挥ArrayList的强悍之处的.

List vs IList

  1. IList是接口，List是对其的实现
  2. 一般内部方法的使用，不考虑是对外暴露，则使用List即可. 如果需要对外暴露，需要对IList重新实现，则必须使用IList
